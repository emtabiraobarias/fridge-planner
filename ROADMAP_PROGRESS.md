**Finding-routing rule:**

| Finding | Fix where | Why |
|---|---|---|
| No scenario/FR covers it (**spec-gap**) | `spec.md` on `main` (+ add scenario to the shared checklist) | Contract gap тАФ both impls owe it; inherited on sync |
| **Frontend** bug | the one branch where it occurs | Vite SPA vs Next SSR genuinely differ |
| **Backend / shared-logic** bug | **both** branches | `packages/server` (Express) is duplicated today тЖТ fix on one, cherry-pick to the other (until C-bis retires Express on `impl/nextjs`) |

**Tip:** walk the same feature area on both branches back-to-back тАФ a failure on both is usually a spec-gap or a shared backend bug; a failure on one is usually frontend-specific.
